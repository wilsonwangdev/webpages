防抖与节流对比

防抖 (Debounce)

防抖是指在事件被触发n秒后再执行回调，如果在这n秒内又被触发，则重新计时。

应用场景

  • 搜索框输入（避免频繁请求）
  • 窗口大小调整
  • 表单验证

节流 (Throttle)

节流是指规定一个单位时间，在这个单位时间内，只能有一次触发事件的回调函数执行。

应用场景

  • 滚动加载
  • 按钮点击（防止重复提交）
  • 鼠标移动事件

演示说明

防抖: 在搜索框中快速输入，只有在停止输入一段时间后才会触发搜索。

节流: 快速点击按钮，无论点击多少次，只会按固定频率处理点击事件。

防抖实现

function debounce(func, wait) {
    let timeout;
    return function() {
        const context = this;
        const args = arguments;
        clearTimeout(timeout);
        timeout = setTimeout(() => {
            func.apply(context, args);
        }, wait);
    };
}

节流实现

function throttle(func, wait) {
    let timeout;
    let previous = 0;
    
    return function() {
        const context = this;
        const args = arguments;
        const now = Date.now();
        
        if (now - previous > wait) {
            func.apply(context, args);
            previous = now;
        }
    };
}

使用示例

// 防抖使用示例
const searchInput = document.getElementById('searchInput');
const searchHandler = () => {
    // 执行搜索操作
    console.log('执行搜索');
};

searchInput.addEventListener('input', debounce(searchHandler, 500));

// 节流使用示例
const button = document.getElementById('clickButton');
const clickHandler = () => {
    // 处理点击事件
    console.log('处理点击');
};

button.addEventListener('click', throttle(clickHandler, 1000));

关键点

1. 防抖和节流都是通过闭包保存状态

2. 防抖使用setTimeout延迟执行，每次触发重置计时器

3. 节流通过时间戳判断是否达到执行间隔

4. 两者都使用apply确保函数执行时的正确上下文
